# W3Css.Blazor 0.3.0

0.3.0 is a non-breaking adoption-confidence release. It keeps the component surface stable and focuses on helping teams install, evaluate, and trust the existing library in app and site work.

## Highlights

- Added a Patterns docs page with four end-to-end examples:
  - App shell dashboard
  - Validated settings form
  - Searchable data table with empty and error states
  - Modal/message-box workflow with action spacing
- Improved README onboarding with a short install path, a first app screen sample, and direct links to key adoption docs.
- Added XML documentation summaries for selected high-use public APIs across theme, layout, forms, data table, overlays, action rows, empty state, menus, and tabs.
- Added a lightweight documentation guard test for selected public API summaries.
- Updated package metadata, changelog, versions docs, and release-quality checks for 0.3.0.

## Compatibility

- No breaking API changes.
- The vendored W3.CSS source remains untouched.
- Consumers should continue to reference `_content/W3Css.Blazor/w3css-blazor.css` for the bundled stylesheet.

## Verification

- `dotnet build W3Css.Blazor.slnx --configuration Release /nr:false`
- `dotnet test W3Css.Blazor.slnx --configuration Release --no-build /nr:false`
- `dotnet pack src/W3Css.Blazor/W3Css.Blazor.csproj --configuration Release --no-build --output artifacts/packages /nr:false`
- `pwsh ./tools/package-consumer-smoke.ps1 -PackageVersion 0.3.0 -PackageSource artifacts/packages`
