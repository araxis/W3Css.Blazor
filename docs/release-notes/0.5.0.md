# W3Css.Blazor 0.5.0

`0.5.0` is a non-breaking starter-kit adoption release. It keeps the component surface stable and focuses on making the package easier to evaluate from a real app shape.

## Highlights

- Added `samples/W3Css.Blazor.StarterKit`, a runnable Blazor WebAssembly app that references the local library project.
- Added dashboard, settings, customers, and workflow routes using the existing app shell, app bar, form, field, data table, empty state, modal, message-box, action-row, and toast primitives.
- Added `/starter-kit` documentation with run commands, expected files, included routes, and related adoption links.
- Replaced the empty product-readiness backlog section on the component index with a starter path.

## Release Quality

- Expanded clean consumer package smoke coverage to compile starter-kit primitives from the package.
- Updated README, Versions docs, changelog, package metadata, and memory files for `0.5.0`.
- Kept the vendored W3.CSS source unchanged.

- `dotnet test W3Css.Blazor.slnx --configuration Release --no-build /nr:false`
