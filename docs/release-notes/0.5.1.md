# W3Css.Blazor 0.5.1

`0.5.1` is a non-breaking starter-kit polish release. It keeps the public component surface stable and focuses on making the starter sample an honest package-consumer shape.

## Changes

- Removed the starter sample's source stylesheet dependency.
- Bundled W3.CSS, theme tokens, and generated package component styles into `_content/W3Css.Blazor/w3css-blazor.css`.
- Improved package-level app shell, app bar, icon, modal, data table, chart, and dark-theme styling used by the starter sample.
- Strengthened chart grid and axis contrast in light and dark themes.
- Hardened package consumer smoke checks with an isolated package cache and bundled stylesheet assertions.
- Updated README, Versions docs, changelog, package metadata, and memory files for `0.5.1`.

## Compatibility

- No breaking API changes.
- No broad new component surface.
- No vendored W3.CSS source edits.

## Verification

- `dotnet build W3Css.Blazor.slnx --configuration Release /nr:false`
- `dotnet test W3Css.Blazor.slnx --configuration Release --no-build /nr:false`
- `dotnet pack src/W3Css.Blazor/W3Css.Blazor.csproj --configuration Release --no-build --output artifacts/packages /nr:false`
- `pwsh ./tools/package-consumer-smoke.ps1 -PackageVersion 0.5.1 -PackageSource artifacts/packages`
- Browser verification covered starter dashboard, customers, settings, and workflow routes.
