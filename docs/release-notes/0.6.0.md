# W3Css.Blazor 0.6.0

`0.6.0` is a non-breaking quality automation release. It keeps the public component surface stable and focuses on making package, docs, and starter-kit regressions harder to ship.

## Quality Automation

- Added `tools/starter-browser-sweep.ps1` for starter-kit browser verification across dashboard, settings, customers, and workflow routes.
- The starter sweep checks console errors, horizontal overflow, route headings, theme toggle cycling, dashboard toast layout, customer search, and workflow modal/message-box open-close behavior.
- Expanded package consumer smoke assertions so the published bundle must contain key package CSS for app shell, nav menu, icons, chart contrast, toast spacing, modal/footer dark contrast, and action-row spacing.

## Release Guardrails

- Strengthened release-quality tests for package version alignment, changelog heading, release notes file, README release link, Versions docs, and release-quality `CurrentVersion`.
- Added release-note command coverage for build, test, pack, local package smoke, docs browser sweep, and starter browser sweep.
- Updated README, Versions docs, changelog, package metadata, starter label, and memory files for `0.6.0`.

## Verification Commands

- `dotnet build W3Css.Blazor.slnx --configuration Release /nr:false`
- `dotnet test W3Css.Blazor.slnx --configuration Release --no-build /nr:false`
- `dotnet pack src/W3Css.Blazor/W3Css.Blazor.csproj --configuration Release --no-build --output artifacts/packages /nr:false`
- `pwsh ./tools/package-consumer-smoke.ps1 -PackageVersion 0.6.0 -PackageSource artifacts/packages`
- `pwsh ./tools/docs-browser-sweep.ps1 -BaseUrl http://localhost:5036 -StartServer`
- `pwsh ./tools/starter-browser-sweep.ps1 -BaseUrl http://localhost:5037 -StartServer`

## Compatibility

- No breaking API changes.
- No broad new component families.
- No vendored W3.CSS edits.
