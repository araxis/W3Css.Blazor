# Memory Index

Last updated: 2026-05-31

This folder is the handoff source for the project. Keep these files current whenever scope, decisions, progress, or verification changes.

To resume in a new session, send: `Resume from memory`.

Resume order:

1. Read `project-memory.md`.
2. Read `current-state.md`.
3. Read the latest dated entries in `progress-log.md`.
4. Use `feature-list.md`, `development-plan.md`, `decisions.md`, `findings.md`, and `w3css-gap-scan.md` for detail as needed.

## Files

- `project-memory.md`: compact project summary and current handoff snapshot.
- `feature-list.md`: canonical feature list and component backlog.
- `development-plan.md`: canonical step-by-step development plan.
- `decisions.md`: architecture, naming, repository, testing, and release decisions.
- `findings.md`: research findings and source notes.
- `w3css-gap-scan.md`: scan of original W3.CSS topics against current library coverage.
- `current-state.md`: current repository state, implemented work, running services, and verification.
- `progress-log.md`: dated progress history.

## Update Rule

When work changes the project, update the owning file and keep `project-memory.md` aligned as the short summary.
